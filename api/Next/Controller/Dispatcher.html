Dispatcher

class Dispatcher extends Object

The Controller Dispatcher reflects over Application Controllers, adding Route Arguments as GET Parameters to their associated Request Object and returning its Response Object with the the contents of the Action Method from the matching Page Controller

Here is also where the Error Standardization Concept takes place, virtually resending the Response by invoking the callback defined within the ControllerException thrown

This handles even nested Exceptions, but it's not bullet proof against infinity recursion created by the developer, redirecting the Response Flow too many times to the point soon or later the start point is reached again

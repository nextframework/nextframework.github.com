Exceptions

InvalidArgumentException Thrown if an Action Method has an malformed Route Definition, with has less than 2 Components â€” a Request Method and a URI
InvalidArgumentException Thrown if an Action Method with a Route defined as a single slash (i.e for a homepage) DO have Route Arguments which is forbidden as for a hierarchy concept
InvalidArgumentException Thrown if there's another Route with exactly the same definition, including the Request Method

Note: that this checking is Controller-aware, which means that Duplicated Routes across different Controller won't be warned. In these cases, both Routes will be recorded but only the one defined earlier will be used as it will be the first match when the chosen Next\HTTP\Router\Router is finding a matching Route

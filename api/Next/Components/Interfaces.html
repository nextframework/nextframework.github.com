Next\Components\Interfaces

Interfaces

Configurable
Configurable Objects are assumed to provide and/or require post-initialization — i.e. after Next\Components\Object::init()`
Contextualizable
Contextualizable Objects allow their context to be extended, almost like PHP 5.4 Traits
Hashable
Hashable Objects provides a string representation of the Object allowing them to be used as array indexes
Informational
Informational Objects provides information messages about their processing state, success, error or just neutral messages
Observer
Observer Interface is part of Observer Design Pattern.
Parameterizable
Parameterizable Objects make use of Next\Components\Parameter` to provide a way of customize classes' functionality with Default, Common and Instance Options
Prototypable
Prototypable Objects prototypes new functionalities to a Prototypical Object, effectively allowing them to be used later — considering the vertically crescent Request Flow, of course
Prototypical
Prototypical Objects allow new functionalities to be prototyped to them -AND- be available to future instances as well — considering the vertically crescent Request Flow, of course — similarly to JavaScript Prototypes
Subject
Subject Interface is part of Observer Design Pattern.

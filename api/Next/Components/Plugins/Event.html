startPropagation()

Starts (or restarts) the propagation of the Event to further Event Listeners.

If multiple Event Listeners are connected to the same Event, and one of Event Listeners stopped the propagation, this allow sequential Listeners to be fired again

This is useful if an Event Handler responsible for a critical error is fired and stopped the propagation, and a complementary, but not crucial, task also fails and needs to inform the endpoint too

stopPropagation()

Stops the propagation of the Event to further Event Listeners.

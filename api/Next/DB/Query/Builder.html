Builder

class Builder extends Object

The Query Builder Object is responsible to assemble a Database Query through class' methods to be then rendered by the Query Renderer associated to the Connection Driver

Builder where(array|string $condition, array|mixed|optional $replacements = [], mixed|string|optional $type = Query::SQL_AND)

Adds a WHERE Clause

Parameters

array|string $condition WHERE Clause
array|mixed|optional $replacements Value for Clause Placeholders, if any

If $condition has multiple possible values, $replacements must be an associative array led by that field name. E.g.:

Array ( [field] => Array ( [0] => value_1 [1] => value_2 [2] => value_3 ) )

This helps not having to manually code a loop calling Build::where() for each one of them with potentially insecure workarounds for the multiplicity
mixed|string|optional $type The WHERE Clause condition type, 'AND' or 'OR'. Defaults to 'AND'

Return Value

Builder Query Builder Object (Fluent Interface)

Exceptions

BadMethodCallException Thrown if WHERE Clause is empty
BadMethodCallException Thrown if trying to create a multiple possibilities condition by defining all of them, repeatedly, as an array, enforcing maintainability
InvalidArgumentException Thrown when creating a multiple possibilities condition without a valid key on the first entry on the replacements array -OR- if this key doesn't match any Clause defined on $condition

Builder join(string|array $table, string $on, string|optional $type = Query::INNER_JOIN)

Adds a JOIN Clause

Parameters

string|array $table A string with the JOIN Table or a single-index array with a JOIN Table and its alias as key:

[ 'm' => 'members' ]

NOTE: In order to implicit maintainability, only the first index will be used. For multiple JOINS, call the method again
string $on The ON Clause
string|optional $type The JOIN Type

The Query Interface has three values of JOIN Types: Query::INNER_JOIN, Query::LEFT_OUTER_JOIN and Query::RIGHT_OUTER_JOIN

However there are no constraints about what it's accepted here because there are numerous valid aliases for this value
